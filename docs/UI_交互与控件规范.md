# UI 交互与控件规范（合并版）

此文档为项目中所有与界面展示、交互控件、触控容错及移动端竖屏体验相关规范的合并版，旨在替代早期分散的多份文档。文档以安卓竖屏（360–430px）为主要目标，并聚焦ND曝光计算器与景深计算器的交互细则。

## 设计原则（摘要）
- 目标设备：安卓移动端竖屏，单手可操作优先。 
- 优先级：可读性 > 操作效率 > 视觉装饰。 
- 主题：默认夜间，提供手动切换到日间（主题变量层实现）。

## 视觉与色彩（Token）
- 夜间（默认）：背景 #0B0D0F，卡片 #111315，强调色 #D6A24A，主文本 #F2F3F4。 
- 日间：背景 #F4F1EA，卡片 #FFFFFF，强调色 #8A5A16，主文本 #1F2328。 
- 使用 CSS 变量映射两套视觉 token，组件不直接引用色值。

## 可复用组件与控件规则
- 顶部：返回/标题/操作（右侧）及紧凑副标题。 
- 选择器（Picker）：水平滚轮样式，中央为当前选中项，左右为前后项（淡入淡出）。
- 刻度滑条（Slider）：用于曝光补偿，中心为 0，步长可配置（常用 0.3EV）。
- 数值卡（Result Card）：主结果大字号、辅助说明小字号。主结果用强调色或高对比文本。
- 按钮：主要 CTA 使用强调色大面积按钮，圆角 14–18px，触控区最小 44x44px。

## 触控与交互细节
- 滑动行为：Picker 需实现 '磁吸'（snap-to-step）和轻微阻尼反馈。松手后自动对齐最近档位。 
- 触觉反馈：每次切换档位时建议触发短促振动（轻量 haptic）。
- 交互态：长曝光计时期间禁用输入区（降低交互透明度并阻止误触）。

## ND曝光计算器（实现要点）

- 摇拍助手（开发中）
- 参数：ND（档位或自定义）、基础快门（预设列表或自定义）、曝光补偿（-3 到 +3 EV）。
- 显示：滤镜后快门为主结果，若 >60s 则使用 `hh:mm:ss` 格式。 
- 交互：ND/快门使用左右 Picker，曝光补偿使用刻度滑条。自定义 ND/快门展开输入框。 
- 无需显示复杂求解目标（固定求解快门），高级参数移至设置或景深页。
 - 注意：界面中原有的“当前 EV100 / EVISO”卡片已从默认 UI 中移除，但该值仍在数据结构与计算中保留（参见 `plan.baseEvIso`）。如需恢复显示，可通过设置面板添加可选开关以控制其可见性。

## 景深计算器（实现要点）

- 放大倍率换算（开发中）
- 参数：画幅（下拉）、焦距（滑条 + 数值）、光圈（滑条）、对焦距离（滑条）。
- 输出：近界（Dn）、远界（Df）、全景深（ΔD）、超焦距（H）。
- 展示：结果以米为单位，遇到无穷远显示 `∞`，内部仍按上限值计算。

## 无障碍与可用性
- 色彩对比满足 WCAG AA（主要文本）。 
- 所有触控目标至少 44x44px，文本具备可缩放支持。 

## 文档与实现注意事项
- 视觉 tokens 由设计文档定义，组件通过 CSS 变量或 theme provider 获取。 
- 所有页面变化应保留向后兼容的 className 与 data-testid，以便测试不因视觉改动而破坏。 

## 实施计划（分步）

- 第一步（当前）：在 `feat/exposure-ux` 分支中提取并实现 CSS 视觉 token（主题变量），并在 `SelectorCard` 中加入左右小三角提示与两侧淡入淡出效果（用于提示可横向滑动）。该步为无侵入样式与轻量交互改动，需运行类型检查与单元测试验证。 
- 第二步：实现 Picker 的磁吸（snap-to-step）与长按加速行为，并为其补充单元测试与 E2E 覆盖。 
- 第三步：为曝光补偿添加 SVG 刻度尺组件并集成触觉反馈（haptic）支持与 reduced-motion 配置。 
- 第四步：视觉微调与可访问性审查（对比度、aria 标注、触控目标）。

**Selector 可调参数**
- **用途**: 用于调优 Exposure 页面中水平 `Picker` / `SelectorCard` 的实时拖拽反馈（位移、微缩、模糊），便于设计与可访问性调整。
- **变量与推荐值**:
	- **`--selector-translate-factor`**: 指针水平位移 (dx) 到元素 `translateX` 的比例。默认 `0.2`。建议范围 `0.08`–`0.4`（较小值更稳定，较大值更明显）。
	- **`--selector-scale-max-shrink`**: 最大缩放收缩量（相对值）。默认 `0.03`（即最多缩小 ~3%）。建议范围 `0`–`0.06`（0 禁用微缩效果）。
	- **`--selector-scale-divisor`**: 影响缩放敏感度的分母（dx / divisor -> shrink）。默认 `800`。建议范围 `200`–`1200`（较小值更快达到最大收缩）。
	- **`--selector-blur-max`**: 最大模糊像素值（单位 px）。默认 `1`。建议范围 `0`–`2`（0 表示禁用模糊）。
	- **`--selector-blur-divisor`**: 影响模糊敏感度的分母（dx / divisor -> blur px）。默认 `200`。建议范围 `100`–`400`。

	- **`--selector-fade-width`**: 边缘淡入淡出覆盖层宽度。默认 `36px`。建议范围 `20px`–`64px`（窄提示到宽提示）。
	- **`--selector-fade-stop`**: 渐变停止位置（百分比），控制渐变紧密度。默认 `12%`。建议范围 `8%`–`30%`（越小渐变越紧密）。
	- **`--selector-fade-opacity`**: 覆盖层基础不透明度。默认 `0.9`。建议范围 `0.6`–`1.0`（较低值更为微妙）。
	- **`--selector-fade-transition`**: 覆盖层过渡时长。默认 `200ms`。建议范围 `120ms`–`300ms`。
	- **`--selector-fade-easing`**: 覆盖层过渡缓动函数。默认 `cubic-bezier(0.2,0.9,0.2,1)`。可使用 `ease` 或自定义 cubic-bezier 获取更弹性的感觉。
- **无障碍与降级**: 在 `prefers-reduced-motion: reduce` 或设计要求下，应将模糊与位移动画降级或禁用（例如把 `--selector-blur-max` 设为 `0`，或完全去掉过渡）。
- **调优建议**: 先在设计工具中以默认值试验，再按需微调 `translate` 与 `scale` 的比例；对低端设备或高频交互场景建议使用更小的 `translateFactor` 与禁用 blur，以节约渲染开销。
