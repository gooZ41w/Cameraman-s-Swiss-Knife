# UI 交互与控件规范（合并版）

此文档为项目中所有与界面展示、交互控件、触控容错及移动端竖屏体验相关规范的合并版，旨在替代早期分散的多份文档。文档以安卓竖屏（360–430px）为主要目标，并聚焦曝光计算器与景深计算器的交互细则。

## 设计原则（摘要）
- 目标设备：安卓移动端竖屏，单手可操作优先。 
- 优先级：可读性 > 操作效率 > 视觉装饰。 
- 主题：默认夜间，提供手动切换到日间（主题变量层实现）。

## 视觉与色彩（Token）
- 夜间（默认）：背景 #0B0D0F，卡片 #111315，强调色 #D6A24A，主文本 #F2F3F4。 
- 日间：背景 #F4F1EA，卡片 #FFFFFF，强调色 #8A5A16，主文本 #1F2328。 
- 使用 CSS 变量映射两套视觉 token，组件不直接引用色值。

## 可复用组件与控件规则
- 顶部：返回/标题/操作（右侧）及紧凑副标题。 
- 选择器（Picker）：水平滚轮样式，中央为当前选中项，左右为前后项（淡入淡出）。
- 刻度滑条（Slider）：用于曝光补偿，中心为 0，步长可配置（常用 0.3EV）。
- 数值卡（Result Card）：主结果大字号、辅助说明小字号。主结果用强调色或高对比文本。
- 按钮：主要 CTA 使用强调色大面积按钮，圆角 14–18px，触控区最小 44x44px。

## 触控与交互细节
- 滑动行为：Picker 需实现 '磁吸'（snap-to-step）和轻微阻尼反馈。松手后自动对齐最近档位。 
- 触觉反馈：每次切换档位时建议触发短促振动（轻量 haptic）。
- 交互态：长曝光计时期间禁用输入区（降低交互透明度并阻止误触）。

## 曝光计算器（实现要点）
- 参数：ND（档位或自定义）、基础快门（预设列表或自定义）、曝光补偿（-3 到 +3 EV）。
- 显示：滤镜后快门为主结果，若 >60s 则使用 `hh:mm:ss` 格式。 
- 交互：ND/快门使用左右 Picker，曝光补偿使用刻度滑条。自定义 ND/快门展开输入框。 
- 无需显示复杂求解目标（固定求解快门），高级参数移至设置或景深页。

## 景深计算器（实现要点）
- 参数：画幅（下拉）、焦距（滑条 + 数值）、光圈（滑条）、对焦距离（滑条）。
- 输出：近界（Dn）、远界（Df）、全景深（ΔD）、超焦距（H）。
- 展示：结果以米为单位，遇到无穷远显示 `∞`，内部仍按上限值计算。

## 无障碍与可用性
- 色彩对比满足 WCAG AA（主要文本）。 
- 所有触控目标至少 44x44px，文本具备可缩放支持。 

## 文档与实现注意事项
- 视觉 tokens 由设计文档定义，组件通过 CSS 变量或 theme provider 获取。 
- 所有页面变化应保留向后兼容的 className 与 data-testid，以便测试不因视觉改动而破坏。
