# 技术框架和技术栈框架

本文档说明项目的实际技术栈、目录结构和工程边界，作为实现和维护的统一参考。

## 1. 实际技术栈

- 前端框架：React 18
- 构建工具：Vite
- 语言：TypeScript
- 状态管理：Zustand
- E2E：Playwright
- 单元测试：Vitest
- 组件测试：@testing-library/react
- 持久化与本地能力：Dexie 已作为依赖保留，用于未来扩展 IndexedDB 场景
- 样式：全局 CSS 文件，不使用 Tailwind 作为当前实现基础

## 2. 脚本与常用命令

- `npm run dev`：启动本地开发服务
- `npm run build`：生产构建
- `npm run preview`：预览构建结果
- `npm run test:unit`：运行单元测试
- `npm run test:e2e`：运行 Playwright E2E

## 3. 分层结构

当前源码采用清晰的三层分工：

- `src/domain/`：纯函数计算与领域数据
- `src/stores/`：页面状态、输入状态和持久化状态
- `src/presentation/`：页面、组件与样式

# 技术框架和技术栈框架

本文档定义项目当前实际采用的技术栈、目录边界、测试约定和变更门槛。

## 1. 实际技术栈

- 前端框架：React 18
- 构建工具：Vite
- 语言：TypeScript
- 状态管理：Zustand
- 单元测试：Vitest
- E2E：Playwright
- 组件测试：@testing-library/react
- 样式：全局 CSS
- 本地存储：Dexie 作为未来 IndexedDB 扩展依赖保留

## 2. 脚本

- `npm run dev`：启动开发服务
- `npm run build`：生产构建
- `npm run preview`：本地预览构建结果
- `npm run test:unit`：运行单元测试
- `npm run test:e2e`：运行 Playwright E2E

## 3. 分层结构

### 3.1 总体分层

```text
UI / presentation
  -> stores
    -> domain
```

### 3.2 职责边界

- `src/domain/`：纯函数计算、公式和静态数据。
- `src/stores/`：页面状态、输入状态和少量跨页面状态。
- `src/presentation/`：页面、组件、布局和样式。

### 3.3 单位规范

- domain 层统一使用可计算单位。
- 景深相关输入和输出以毫米为主，UI 可以转成米显示。
- 色温使用 Kelvin。

## 5. 当前实现对应关系

- 曝光页面：`src/presentation/screens/ExposureScreen.tsx`
- 景深页面：`src/presentation/screens/DofScreen.tsx`
- 全局壳：`src/presentation/components/AppShell.tsx`
- 容差显示：`src/presentation/components/ToleranceBadge.tsx`
- 免责声明：`src/presentation/components/DisclaimerFooter.tsx`
- 色温预览逻辑：`src/presentation/components/AppShell.tsx` + `src/domain/temperature.ts`

## 6. 设计原则

- 领域层禁止依赖 DOM。
- 页面层只负责交互编排和展示格式化。
- 状态层不写核心公式。
- 任何公式变更都必须先落到 domain，再同步测试与文档。

## 7. 测试门槛

- 关键公式必须有单元测试。
- 关键页面必须有 E2E 回归测试。
- 修改阈值、单位或结果格式时，必须同时更新测试断言。

## 8. 变更控制

以下内容变更时，必须同步更新代码、测试和文档：

- 公式
- 阈值
- 单位口径
- 结果格式
- 页面入口和导航
